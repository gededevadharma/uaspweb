BAGIAN 1: ARSITEKTUR APLIKASI

1.1 Pola MVC (Model-View-Controller)

Aplikasi ini menggunakan framework CodeIgniter 3 dengan pola arsitektur MVC:

KomponenFungsiLokasi Folder
ModelLogika database, query SQL, manipulasi dataapplication/models/
ViewTemplate HTML, tampilan antarmuka penggunaapplication/views/
ControllerPenghubung Model & View, logika bisnis, routing requestapplication/controllers/

Alur Request:

Browser → URL → Router (application/config/routes.php)
       → Controller (memproses logika bisnis)
       → Model (jika perlu data database)
       → View (render HTML)
       → Browser (menampilkan hasil)

DDL (Data Definition Language)

CREATE TABLE buku (
    id INT(11) AUTO_INCREMENT PRIMARY KEY,
    judul VARCHAR(200) NOT NULL,
    pengarang VARCHAR(150),
    penerbit VARCHAR(150),
    isbn VARCHAR(20) UNIQUE,
    tahun INT(4),
    jumlah INT(5) DEFAULT 1,
    tersedia INT(5) DEFAULT 1,
    deskripsi TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE peminjam (
    id INT(11) AUTO_INCREMENT PRIMARY KEY,
    nama VARCHAR(100) NOT NULL,
    no_hp VARCHAR(20),
    alamat TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE peminjaman (
    id INT(11) AUTO_INCREMENT PRIMARY KEY,
    peminjam_id INT(11) NOT NULL,
    buku_id INT(11) NOT NULL,
    tgl_pinjam DATE NOT NULL,
    tgl_kembali DATE NOT NULL,
    tgl_dikembalikan DATE,
    status ENUM('dipinjam','dikembalikan') DEFAULT 'dipinjam',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (peminjam_id) REFERENCES peminjam(id) ON DELETE CASCADE,
    FOREIGN KEY (buku_id) REFERENCES buku(id) ON DELETE CASCADE
);

2.3 Model & Query Utama

Buku_model

MethodSQL
get_all()SELECT * FROM buku ORDER BY created_at DESC LIMIT ? OFFSET ?
count_all()SELECT COUNT(*) FROM buku WHERE ...
kurangi_tersedia($id)UPDATE buku SET tersedia = tersedia - 1 WHERE id = ? AND tersedia > 0
tambah_tersedia($id)UPDATE buku SET tersedia = tersedia + 1 WHERE id = ?

Peminjaman_model — Query Join Utama

$this->db->select('peminjaman.*, 
    peminjam.nama as nama_peminjam, 
    peminjam.no_hp, 
    peminjam.alamat, 
    buku.judul as judul_buku, 
    buku.pengarang, 
    buku.penerbit, 
    buku.isbn');
$this->db->from('peminjaman');
$this->db->join('peminjam', 'peminjam.id = peminjaman.peminjam_id', 'left');
$this->db->join('buku', 'buku.id = peminjaman.buku_id', 'left');
$this->db->order_by('peminjaman.created_at', 'DESC');

// Filter terlambat:
if ($status == 'terlambat') {
    $this->db->where('peminjaman.status', 'dipinjam');
    $this->db->where('peminjaman.tgl_kembali <', date('Y-m-d'));
}

Fitur Terlambat (Logika)

Deteksi keterlambatan dilakukan di View dengan PHP:

$terlambat = ($p->status == 'dipinjam' && strtotime($p->tgl_kembali) < time());
// Jika true → badge merah "Terlambat" + baris table-danger

d) Search & Pagination

Search: Input teks → GET parameter ?search=... → Controller filter via LIKE query

Pagination: Parameter ?page=N → LIMIT 10 OFFSET (page-1)*10

e) Filter Status (Peminjaman)

Tombol filter: ?status=dipinjam, ?status=dikembalikan, ?status=terlambat

Untuk status terlambat, filter dilakukan di model dengan kondisi:

WHERE peminjaman.status = 'dipinjam' 
  AND peminjaman.tgl_kembali < CURDATE()

f) Detail Modal (View Peminjaman)

Tombol 👁️ (View) pada setiap baris peminjaman membuka modal yang menampilkan:

  • Data Peminjam: Nama, No. HP, Alamat
  • Data Buku: Judul, Pengarang, Penerbit, ISBN
  • Data Peminjaman: Tgl Pinjam, Tgl Kembali, Tgl Dikembalikan, Status

Deteksi otomatis: Jika hari ini > tgl_kembali dan status masih "dipinjam":

  • Filter khusus "Terlambat" untuk melihat semua peminjaman yang melewati batas

BAGIAN 5: ROUTING & KONFIGURASI

5.1 Routes (config/routes.php)

$route['default_controller'] = 'auth/login';

// Auth
$route['login']    = 'auth/login';
$route['logout']   = 'auth/logout';

// Dashboard
$route['dashboard'] = 'dashboard/index';

// Buku
$route['buku']                = 'buku/index';
$route['buku/tambah']         = 'buku/tambah';
$route['buku/edit/(:num)']    = 'buku/edit/$1';
$route['buku/hapus/(:num)']   = 'buku/hapus/$1';
$route['buku/detail/(:num)']  = 'buku/detail/$1';

// Peminjaman
$route['peminjaman']                = 'peminjaman/index';
$route['peminjaman/tambah']         = 'peminjaman/tambah';
$route['peminjaman/kembali/(:num)'] = 'peminjaman/kembali/$1';

Catatan: Admin menggunakan prefix admin/ → contoh: admin/buku → controller admin/Buku.php. Namun karena tidak ada aturan routing eksplisit untuk admin, CodeIgniter secara otomatis menangani sub-direktori controller.

$config['base_url'] = ((isset($_SERVER['HTTPS']) && $_SERVER['HTTPS'] == 'on')
